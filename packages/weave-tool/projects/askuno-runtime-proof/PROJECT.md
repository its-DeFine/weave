# Askuno Runtime Proof

Askuno Runtime Proof is the first admitted WEAVE worked example. It records how
the wrapper guides an execution runtime through intent, research, selection,
planning, engineering, QA, KPI setup, marketing, and the parallel
iteration-analysis loop.

The active marketing wedge is the accountless Askuno API trial: an owner-gated
GitHub and Discord launch plan where humans or agents can press "Use now,"
start with no account and no money required, receive $10 of free API credit,
and then be invited to claim or top up the workspace after trial usage creates
enough context.

The project must proceed through lifecycle gates. Engineering work depends on
Research admission. QA depends on Engineering evidence. Marketing and the local
iteration-analysis loop both start from KPI setup. External distribution remains
owner-gated; local iteration planning can run while marketing evidence arrives.
